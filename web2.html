// CS101 · Computer Organization

Computer
Organization

From transistors to CPU pipelines — understand the architecture that powers every computation.

CONCEPT 01
Von Neumann Architecture
The classical model where a single memory stores both program instructions and data. Key components: CPU, Memory, I/O, and the Bus connecting them. The CPU fetches, decodes, and executes instructions in a cycle.
CONCEPT 02
CPU Components
The CPU contains the ALU (arithmetic/logic), CU (control unit), and Registers. Key registers: Program Counter (PC), Instruction Register (IR), Memory Address Register (MAR), and Memory Buffer Register (MBR).
CONCEPT 03
Memory Hierarchy
Registers → Cache (L1/L2/L3) → RAM → Secondary Storage. Faster = smaller and more expensive. The locality of reference principle means frequently used data is kept in faster memory.
⚙️Instruction Execution Cycle
▼

The Fetch-Decode-Execute (FDE) cycle is the fundamental operation of a CPU:

1. Fetch

  • Copy address from PC to MAR
  • Read instruction from memory into MBR
  • Copy MBR to IR, increment PC

2. Decode

  • Control Unit interprets the instruction in IR
  • Identifies the operation and operands

3. Execute

  • ALU performs the operation
  • Results stored in registers or memory
PC → MAR → Memory → MBR → IR → Decode → Execute
🔢Number Systems & Data Representation
▼

Computers work in binary (base-2). Understanding conversions is essential:

SystemBaseDigitsExample
Binary20,11010₂ = 10
Octal80–712₈ = 10
Decimal100–910
Hexadecimal160–9, A–FA₁₆ = 10
Binary to Decimal: 1010₂ = 1×2³ + 0×2² + 1×2¹ + 0×2⁰ = 10

Two's Complement (for signed integers)

  • Invert all bits, then add 1
  • Example: −5 in 8-bit = 11111011
  • Range: −128 to +127 for 8-bit
🚀Pipelining
▼

Pipelining overlaps multiple instruction cycles to improve throughput. Like an assembly line — while one instruction executes, the next is being decoded, and the one after is being fetched.

Speedup = Time without pipeline / Time with pipeline

Hazards in Pipelining

  • Data Hazard: Instruction needs result of previous instruction
  • Control Hazard: Branch instructions disrupt pipeline flow
  • Structural Hazard: Two instructions need same hardware resource

// CS102 · Programming Fundamentals

Programming
Fundamentals

Master the building blocks of software — logic, control flow, functions, and data structures.

CONCEPT 01
Variables & Data Types
Variables are named storage locations. Types include int, float, char, bool, and string. Type determines the size and kind of data stored and operations allowed.
CONCEPT 02
Control Flow
Programs don't always run top-to-bottom. if/else, switch, for, while, and do-while structures control execution order based on conditions and repetitions.
CONCEPT 03
Functions
Functions are reusable blocks of code. They have a return type, name, and optional parameters. Functions promote DRY (Don't Repeat Yourself) programming and modularity.
📦Arrays & Data Structures
▼

An array is a fixed-size collection of same-type elements stored in contiguous memory.

int arr[5] = {10, 20, 30, 40, 50}; // arr[0] = 10

Key Data Structures

  • Array: Random access, O(1); insert/delete O(n)
  • Stack: LIFO — Last In, First Out. Push/Pop operations
  • Queue: FIFO — First In, First Out. Enqueue/Dequeue
  • Linked List: Dynamic size, nodes with data + pointer

🌀Recursion
▼

A function that calls itself. Every recursive solution needs a base case (stops recursion) and a recursive case.

factorial(n) = n × factorial(n-1), base: factorial(0) = 1

When to use recursion

  • Problems that can be broken into smaller similar sub-problems
  • Tree/graph traversal, divide-and-conquer algorithms
  • Warning: deep recursion can cause stack overflow

// EE101 · Digital Systems

Digital
Systems

The language of electronics — binary logic, gates, and circuits that make computers possible.

CONCEPT 01
Boolean Algebra
Mathematical framework for binary logic. Operations: AND (·), OR (+), NOT ('). Laws include commutative, associative, distributive, and De Morgan's theorems.
CONCEPT 02
Logic Gates
Physical implementation of Boolean operations. Basic gates: AND, OR, NOT, NAND, NOR, XOR, XNOR. NAND and NOR are universal gates.
CONCEPT 03
Sequential Circuits
Circuits with memory — output depends on current input AND past state. Built using flip-flops. Examples: registers, counters, shift registers, and finite state machines.
📊Truth Tables & Boolean Expressions
▼

Truth tables enumerate all possible input combinations and their outputs.

De Morgan's Theorems

NOT(A AND B) = NOT(A) OR NOT(B)
NOT(A OR B) = NOT(A) AND NOT(B)
🔁Flip-Flops
▼

Flip-flops are 1-bit memory elements triggered by clock signals.

Types of Flip-Flops

  • SR Flip-Flop: Set-Reset. Forbidden state when S=R=1
  • D Flip-Flop: Data flip-flop. Q follows D on clock edge. Most common
  • JK Flip-Flop: Like SR but no forbidden state. J=K=1 toggles output
  • T Flip-Flop: Toggle. Output flips when T=1 on clock edge
D Flip-Flop: Q(next) = D (on rising clock edge)
🗺️Karnaugh Maps (K-Maps)
▼

A visual method to simplify Boolean expressions by grouping 1s in the truth table.

Rules for K-Map grouping

  • Groups must be powers of 2 (1, 2, 4, 8…)
  • Groups can wrap around edges
  • Larger groups = more simplification
  • Each cell can be in multiple groups
Goal: Find the Minimum Sum of Products (SOP) or Product of Sums (POS) expression

// EN101 · English

English
Language

Professional communication skills for the IT world — writing, speaking, and presenting with clarity.

CONCEPT 01
Technical Writing
Clear, concise, and accurate writing for technical audiences. Includes documentation, reports, manuals, and proposals. Key: know your audience and purpose.
CONCEPT 02
Grammar Essentials
Master tenses, subject-verb agreement, articles, prepositions, and punctuation. Correct grammar builds credibility and ensures clarity in professional contexts.
CONCEPT 03
Presentations & Speaking
Structure: Introduction → Body → Conclusion. Use signposting language, maintain eye contact, manage pace and tone. PowerPoint best practices for technical presentations.
✍️Essay & Report Writing
▼

Report Structure

  • Title Page: Title, author, date, institution
  • Abstract/Executive Summary: Brief overview (150-200 words)
  • Introduction: Background, purpose, scope
  • Body/Findings: Detailed analysis with headings
  • Conclusion: Summary and recommendations
  • References: IEEE or APA citation format
"Good writing is clear thinking made visible." — Clarity over complexity always.

// MM101 · Multimedia

Multimedia
Systems

The art and science of digital media — from pixels and sound waves to streaming video and interactive design.

CONCEPT 01
Digital Images
Images are grids of pixels. Resolution = width × height in pixels. Color depth determines colors per pixel (8-bit = 256 colors, 24-bit = 16.7M colors). File size = Resolution × Color Depth.
CONCEPT 02
Audio Fundamentals
Sound is captured via sampling (samples per second = Hz) and quantization (bit depth). CD quality: 44.1 kHz, 16-bit. Higher values = better quality but larger files.
CONCEPT 03
Video & Compression
Video = sequence of frames (images). Frame rate (fps) determines smoothness. Compression removes redundancy — spatial (within frames) and temporal (between frames). Standards: H.264, H.265.

📁File Formats & Compression
▼

Image Formats

  • JPEG: Lossy compression, best for photos (DCT-based)
  • PNG: Lossless, supports transparency (good for graphics)
  • GIF: 256 colors, supports animation
  • SVG: Vector format, scalable without quality loss

Audio/Video Formats

  • MP3: Lossy audio (MPEG Layer 3), perceptual coding
  • WAV: Uncompressed audio, high quality, large files
  • MP4: Container for H.264 video + AAC audio
  • WebM: Open web video format using VP8/VP9
Compression Ratio = Original Size / Compressed Size
🌐Multimedia on the Web
▼

Web multimedia must balance quality with loading performance.

Key Concepts

  • Streaming: Progressive delivery — playback before full download
  • CDN: Content Delivery Networks distribute media geographically
  • Responsive Images: Different sizes for different screen resolutions
  • Lazy Loading: Load media only when it enters viewport
Rule of thumb: Use the smallest file that maintains acceptable quality for the target audience.
